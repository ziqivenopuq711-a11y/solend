TypeScript / JavaScript

SDK & Libraries

The official @solendprotocol/solend-sdk package provides a fully typed TypeScript client for interacting with the Solend protocol. Available on npm, the SDK abstracts all the complexity of constructing Solana transactions, deserializing on-chain account state, and computing derived metrics like APYs and health factors.

The SDK is actively maintained and versioned with full changelog documentation. It supports both browser and Node.js environments, making it suitable for frontends, backend services, bots, and CLI tooling. All public methods are fully typed with JSDoc documentation for IDE autocomplete support.

✓ Full TypeScript types for all protocol structs
✓ Compute APYs, borrow limits, and health factors
✓ Build supply, borrow, repay, and withdraw transactions
✓ Flash loan transaction builder included
✓ Works with Anchor-compatible wallet adapters

# Install the SDK

npm install @solendprotocol/solend-sdk

// Initialize the SDK and fetch reserve data

import { SolendMarket } from '@solendprotocol/solend-sdk';


const market = await SolendMarket.initialize(

connection,

"production"

);


// Fetch all reserves

const reserves = await market.fetchReserves();


// Get USDC supply APY

const usdc = reserves.find(

r => r.config.symbol === "USDC"

);

console.log(usdc.supplyAPY); // e.g. 0.0618

REST API

API Reference

Solend's public REST API provides read access to all protocol state, historical data, and user positions. No API key required for public endpoints.

GET /v1/markets

Returns all active lending markets with their configuration, supported assets, TVL, total borrows, and current utilization rates. Updated every block.

GET /v1/reserves

Fetches detailed reserve data for all supported assets including supply APY, borrow APY, collateral factor, liquidation threshold, and current exchange rate.

GET /v1/obligations/:wallet

Returns a specific wallet's open lending positions — supplied assets, borrowed assets, health factor, and maximum borrowable amount across all markets.

GET /v1/rates/history

Historical interest rate data for all reserves, queryable by asset, time range, and granularity (hourly, daily, weekly). Useful for building APY charts and rate analytics.

GET /v1/liquidations

Lists recent liquidation events with repaid amounts, received collateral, liquidator addresses, and transaction signatures for transparency and analytics.

GET /v1/stats/tvl

Time-series TVL data for the protocol and individual markets. Supports filtering by asset, market, and date range. Used by DeFiLlama and other analytics aggregators.

On-Chain Integration

Smart Contract CPI Integration

Solend's lending program supports Cross-Program Invocation (CPI), allowing other Solana programs to call Solend's instructions directly from their own program logic. This enables fully on-chain integrations where your protocol can supply, borrow, repay, and withdraw from Solend atomically within your own transactions.

The CPI interface is stable and versioned. All Solend instructions are available as CPI entry points, including supply, borrow, repay, withdraw, liquidate, and flash loan. Using Anchor's CPI clients, integration is straightforward and type-safe.

Solend also provides a dedicated CPI crate on crates.io for Rust programs. The crate is updated alongside the main program and provides struct definitions, instruction builders, and helper functions for common integration patterns used by vault protocols and automated strategy programs.

// Rust CPI example — borrow from Solend

use solend_cpi::cpi::borrow_obligation_liquidity;


pub fn my_borrow(

ctx: Context<MyBorrow>,

amount: u64,

) -> Result<()> {

borrow_obligation_liquidity(

CpiContext::new(

ctx.accounts.solend_program.to_account_info(),

BorrowObligationLiquidity {

source_liquidity: ctx.accounts.reserve_liquidity.to_account_info(),

destination_liquidity: ctx.accounts.dest.to_account_info(),

// ... other accounts

},

),

amount,

)?;

Ok(())

}
